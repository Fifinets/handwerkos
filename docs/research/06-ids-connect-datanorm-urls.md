### Hinweis: Kein oeffentliches GitHub-Repo oder OpenAPI-Spec

Es gibt **keine** oeffentliche OpenAPI-Spezifikation oder GitHub-Repository fuer IDS-Connect. Die Spezifikation ist als PDF bei ITEK frei verfuegbar (siehe oben). IDS-Connect ist ein HTTP-basiertes Protokoll (URL-Parameter + HTML-Form-Posts), kein modernes REST-API.

### Wichtig: Datanorm ist KEIN offener Standard

Die Spezifikation ist **nicht frei verfuegbar**. Es gibt keine Open-Source-Implementierung der offiziellen Spec. Die informelle Struktur ist aber bekannt:
- Multi-CSV Textdateien (Codepage 850)
- Satzarten: A (Artikeldaten), B (Bestelldaten), P (Preise), R (Rabatte)
- Version 4.0: Artikelstammdaten + Preise + Rohstoffzuschlaege + Zeiten
- Version 5.0: + Grafikanbindung + erweiterte Zeichen + Fliesendaten

## Zusammenfassung: Prioritaet fuer HandwerkOS-Integration

### Phase 1 (Datanorm Import - bereits geplant)
1. **IDS Spec PDF herunterladen:** https://www.itek.de/fileadmin/user_upload/itek-website/Wissen/Standardisierung/IDS_Connect/2.5/20201225_IDS_Schnittstelle_2_5.pdf
2. **Datanorm Parser als Referenz:** https://github.com/FahrJo/datanorm-python (Python, aber gute Referenz fuer Satzarten A/B/P/R)
3. **Datanorm Versionen verstehen:** https://www.datanorm.de/DATANORM/DATANORM-Versionen
4. **Grosshaendler-Datanorm testen:** Sonepar (digitalsupport@sonepar.de) oder Rexel

### Phase 2 (ETIM Klassifikation)
1. **ETIM API Zugang beantragen:** info@etim-international.com (Betreff: "Request_clientId")
2. **ETIM API testen:** https://etimapi.etim-international.com/swagger/index.html
3. **REST Client:** https://bitbucket.org/3xt/etimapi-rest-client/src/master/

### Phase 3 (IDS-Connect Live)
1. **IDS Spec studieren** (PDF oben)
2. **Bei ITEK registrieren** als Softwarehersteller
3. **Mit Sonepar/Rexel Testumgebung vereinbaren**

### Phase 4 (GAEB Export)
1. **GAEB Schema laden:** https://www.gaeb.de/en/service/downloads/gaeb-dataexchange/
2. **Fachdoku lesen:** https://www.gaeb.de/wp-content/uploads/2023/09/Fachdokumentation_GAEB-DA-XML_3.3_2023-01.pdf
3. **gaeb4linux als Referenz:** https://github.com/klaus4772/gaeb4linux
